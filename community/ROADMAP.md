# HERALD Roadmap

## v1.0 — Current (draft)

- ✅ Core spec: manifest + 5 layers
- ✅ Progressive disclosure (Tier 1/2/3)
- ✅ JSON schemas
- ✅ Examples: Next.js, FastAPI, Go
- ✅ Compatibility: AGENTS.md, CLAUDE.md, SKILL.md
- ✅ Install script
- ✅ Interactive generator (herald.sh)

## v1.1 — Near term

- [ ] `herald compile` CLI — generate vendor files from `.herald/`
- [ ] Skill registry — reference shared skills by URL in `main.yaml`
- [ ] VS Code extension — validate `.herald/` against schemas
- [ ] More examples: Rails, Django, Rust/Axum, Laravel, .NET

## v2.0 — Future

- [ ] Workflow layer — multi-step agent orchestration
- [ ] Monorepo support — `herald.workspace.yaml` at root
- [ ] Hook system — pre/post actions triggered by agent events
- [ ] Profile system — swap context sets per environment

## How to influence the roadmap

Open a GitHub issue with the `roadmap` label. Describe your use case and why it matters. The most compelling real-world use cases move up the priority list.
