# Project Context

## Overview
Full-stack Next.js 14 application using App Router. Serves a SaaS product for project management.

## Architecture
- **Frontend**: Next.js 14 App Router, React Server Components, Tailwind CSS
- **API**: tRPC v11 for type-safe client–server communication
- **Database**: PostgreSQL via Prisma ORM
- **Auth**: NextAuth.js v5 with Google + GitHub providers
- **Deployment**: Vercel (frontend + API routes) + Supabase (database)

## Stack
- TypeScript 5.4 (strict mode)
- Next.js 14 (App Router)
- tRPC v11
- Prisma 5
- Tailwind CSS + shadcn/ui
- Vitest + React Testing Library
- Zod for all validation

## Project structure
```
src/
├── app/                  # Next.js App Router pages and layouts
│   ├── (auth)/           # Auth-related routes
│   ├── (dashboard)/      # Protected dashboard routes
│   └── api/trpc/         # tRPC API handler
├── components/           # Shared React components
│   ├── ui/               # shadcn/ui primitives (do not modify directly)
│   └── features/         # Feature-specific components
├── server/
│   ├── api/              # tRPC routers
│   └── db/               # Prisma client and database utilities
├── lib/                  # Shared utilities and helpers
└── types/                # Global TypeScript types
```

## Conventions
- **Named exports only** — no default exports except page.tsx and layout.tsx
- **Server Components by default** — add `"use client"` only when necessary
- **Zod everywhere** — all external input validated with Zod schemas
- **tRPC for all data fetching** — no raw fetch() calls to the API
- **Tailwind classes** — no inline styles, no CSS modules
- **Error handling** — use `TRPCError` in procedures, never throw raw errors
- **File naming** — kebab-case for files, PascalCase for components

## Testing
- Unit tests: Vitest with `*.test.ts` co-located with source
- Integration tests: Vitest with `*.integration.test.ts` in `tests/`
- Component tests: React Testing Library, never test implementation details
- Run: `pnpm test` (watch mode), `pnpm test:ci` (single run)

## Do Not Touch
- `src/components/ui/` — auto-generated by shadcn, update via CLI only
- `prisma/migrations/` — never edit manually, use `prisma migrate dev`
- `.env.production` — never read in application code, only in CI/CD
- `src/app/api/auth/` — NextAuth internals, modify only with extreme care
